# docs/knowledge

This directory holds extracted text and Markdown converted from source PDFs and requirements documents.

## Purpose

Provides human-readable, version-controlled copies of reference material used to inform ticket writing — including product requirement docs, onboarding guides, and knowledge bases.

## Naming Convention

- Use **kebab-case** for all filenames.
- For versioned or dated documents, **prefix with the date** in `YYYY-MM-DD` format:
  - e.g. `2024-03-04-gem-instructions.md`
- For stable/evergreen documents, omit the date prefix:
  - e.g. `aacf-knowledge-document.md`
